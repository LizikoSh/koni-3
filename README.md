# Кінь.Світ

Адаптивний українськомовний статичний сайт про коней, готовий до публікації на **GitHub Pages**.

## Що є в проєкті

- енциклопедія з 30 породами;
- пошук і фільтри за регіоном, типом та призначенням;
- вибране збережається в `localStorage`;
- порівняння 2–3 порід;
- модальні картки з розширеною довідкою;
- художні історії та авторська поезія;
- форма власної історії зі збереженням чернетки й експортом у `.txt`;
- мінісловник кінних термінів;
- світла й темна тема;
- адаптація під смартфони;
- жодних фреймворків або збірки.

## Файли

```text
kin-svit/

```

## Як опублікувати на GitHub Pages

1. Створіть новий публічний репозиторій на GitHub.
2. Завантажте всі файли з папки `kin-svit` у корінь репозиторію.
3. Відкрийте **Settings → Pages**.
4. У блоці **Build and deployment** виберіть:
   - Source: `Deploy from a branch`;
   - Branch: `main`;
   - Folder: `/ (root)`.
5. Натисніть **Save** і зачекайте кілька хвилин.

Адреса матиме приблизний вигляд:

```text
https://ВАШ-ЛОГІН.github.io/НАЗВА-РЕПОЗИТОРІЮ/
```

## Як додати нову породу

Відкрийте `data.js` і додайте новий об’єкт у масив `breeds` за існуючим зразком. Значення `id` має бути унікальним.

## Зображення

Макет використовує віддалені ілюстративні фото з Unsplash. Для повноцінного редакційного сайту краще:

- замінити фото на точно ідентифіковані зображення конкретних порід;
- перевірити ліцензію кожного файлу;
- додати автора, джерело та умови використання;
- за потреби завантажити оптимізовані `.webp` у локальну папку `images/`.

## Важливо про контент

Довідкові картки є стартовим редакційним матеріалом. Перед публікацією як фахової енциклопедії варто звірити дані з офіційними племінними книгами, асоціаціями порід і міжнародними базами.
